Chat Interface Flow

User visits chat page —> System initializes with default settings —> User uploads documents (PDF, PPT, DOCX, images) or starts conversation without documents —> Documents processed: PDFs chunked via RecursiveCharacterTextSplitter with metadata embedding using Ollama embeddings —> Embeddings stored/retrieved from Chroma vector database —> User submits query —> System performs AI classification (GREETING/SIMPLE/RAG) using Ollama DeepSeek R1 —> For RAG queries: Query embedding generated and searched against document vectors —> Context retrieved from relevant document chunks —> AI generates response using Ollama DeepSeek R1 —> Response displayed to user with streaming capability —> Sources cited when documents used —> Chat history maintained for conversation context
